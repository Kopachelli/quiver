# Privacy Policy — Quiver

_Last updated: 2026-06-06_

**Quiver does not collect, transmit, store, or share any personal data.** It is a local Windows
desktop application that runs entirely on your device.

## What Quiver does

- **No telemetry, no analytics, no tracking.** Quiver contains no usage tracking, no analytics
  SDKs, and no remote crash reporting.
- **No network connections.** Quiver makes no internet or server connections. It does not "phone
  home," check for updates online, or send anything anywhere.
- **Local files only.** Quiver reads and edits the skill, MCP‑server, and plugin files that your
  AI coding agents already keep in your home directory (e.g. `%USERPROFILE%\.cursor`, `\.claude`,
  `\.codex`, `\.hermes`, `\.pi`, `\.openclaw`). These files never leave your computer.
- **Settings & logs stay local.** Your settings are saved at `%APPDATA%\Quiver\settings.json`.
  If an unexpected error occurs, a diagnostic note may be written to `%APPDATA%\Quiver\crash.log`.
  Neither is ever transmitted.
- **No accounts, no sign‑in, no cookies.**

The only actions that involve other programs are ones you explicitly trigger — e.g. "Reveal in
Explorer", "Open in Default App", or "Open in Cursor" — which launch local applications on your PC.

## The website and downloads

The Quiver website (`kopachelli.github.io/quiver`) is hosted on **GitHub Pages**, and downloads are
served from **GitHub Releases**. When you visit those, GitHub may process standard request
information (such as your IP address and browser user agent) under
[GitHub's Privacy Statement](https://docs.github.com/site-policy/privacy-policies/github-general-privacy-statement).
This is GitHub's processing, not Quiver's.

## Changes

If this policy changes, the updated version will be published here. Material changes will be noted in
the release notes.
